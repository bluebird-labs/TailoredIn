How inline editing works

  • Click any section (title, metadata, summary, accomplishment) to enter edit mode
  • The markdown structure stays visible — #, ---, >, - markers remain during editing
  • Frontmatter becomes editable fields with key: value layout preserved
  • Active field gets a subtle amber highlight row
  • Dashed underlines on values indicate editability, solidify on focus
  • No modal, no context switch — you edit exactly where you read
  • Save bar appears at the bottom with dirty count + Save/Discard

Key design decisions

  • Section-level editing — click a section to edit just that part (preserves mutual exclusion from current UX)
  • Or full-document editing — shown here, the entire experience becomes editable at once (more Obsidian-like)
  • Markdown is the frame — the structure markers are not just decorative, they define the edit affordances
  • No form fields — inputs are minimal, unstyled, flowing within the markdown layout
  • The + add accomplishment prompt follows markdown list convention
  • Milkdown editor would handle the summary/narrative fields for rich markdown editing
