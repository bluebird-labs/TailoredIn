What changed

  • # page heading with frontmatter-style meta (count:, directory:)
  • Search field uses / prefix (vim/markdown search convention) instead of magnifying glass icon
  • ## section header for results with entry count
  • Each card title has ### marker — heading hierarchy mirrors a markdown document
  • Domain name shown in monospace below company name (replaces badge clutter)
  • Card metadata rendered as inline frontmatter: key: value pairs on one line
  • Description as blockquote with > marker
  • Badges replaced by inline key: value metadata — cleaner, more informational
  • + new button in monospace — terse, action-oriented

Design rationale

  • Heading hierarchy: # page → ## section → ### card. The whole page IS a markdown document
  • Metadata over badges: YAML-style key: value is more informative than colored pills — you see what each value means
  • / search: Familiar to vim, Obsidian, and Notion users. Reinforces the "document" feel
  • Domain as identity: The monospace domain name below the company name is more useful than a URL badge
  • Blockquote descriptions: The > marker gives descriptions visual weight and structure
  • Consistent amber: All markers (#, /, >, key:) use the primary amber color
