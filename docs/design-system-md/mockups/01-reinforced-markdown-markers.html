# What changed from v1

  • # heading marker is now full-size (22px), amber-colored, same weight as heading text
  • --- frontmatter fences are visible as amber delimiters wrapping a code-bg block
  • key: labels in frontmatter use amber-muted color, creating a YAML-like feel
  • > blockquote marker is visible, amber, and the block has a warm tinted background
  • - list bullets are larger (15px), amber, monospace — unmistakably markdown
  • ## section markers match the amber heading treatment
  • Breadcrumb separator is now / instead of › — filesystem-like

✦ The amber connection

  • All markdown markers use the primary amber color — they ARE the brand
  • Frontmatter block has warm background tint from the amber palette
  • Blockquote left border is 3px solid amber
  • Code-bg badges have amber border accent
  • The markers are not decoration — they're the primary visual identity
  • When someone sees this page, the amber markdown syntax IS the first thing they notice
