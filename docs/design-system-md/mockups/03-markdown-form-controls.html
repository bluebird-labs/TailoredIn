Markdown Form Controls

Every non-text control rendered as if it belongs in a markdown document. View mode reads like YAML. Edit mode stays in the flow.

Select / Dropdown e.g. company link, industry
View
---
company: Voi Technology
industry: Micromobility
---
Edit
---
company: Voi Technology
industry: Micromobility ▼
✓ Micromobility
  Microservices
  Microelectronics
---
Pattern: In view mode, the value renders as plain text — indistinguishable from a string field. In edit mode, clicking the value reveals a ▼ chevron and opens a searchable dropdown. The dropdown uses the same code-bg styling. Values that come from a fixed set get a subtle chevron affordance on hover.
Checkbox / Toggle e.g. featured, remote, current role
View
---
remote: [x] true
featured: [ ] false
current: [x] true
---
Edit
---
remote: ✓ true
featured: false
current: ✓ true
---
Pattern: Borrows GitHub-flavored markdown task syntax. View mode shows [x] and [ ] as monospace text with true/false. Edit mode swaps to a real checkbox with amber border/fill but keeps the true/false label inline. Single click toggles. The GFM checkbox is instantly recognizable to anyone who's used markdown.

Pattern: Dates render as ISO-style YYYY-MM — the most "markdown/YAML-native" format. In edit mode, clicking the date splits it into year and month segments. Clicking a segment opens a compact month picker. The picker drops from the value position, not from a separate icon. The date string itself IS the trigger. Year navigation with ‹ › arrows.
Numeric Range (+/-) e.g. bulletMin, bulletMax
View
---
bullets: 2–5
---
Edit
---
bullets: − 2 + to − 5 +
---
Pattern: View mode shows the range as a compact min–max string. Edit mode reveals − / + stepper buttons flanking each number. The buttons use monospace font, amber color, and minimal styling (border, no fill). The to separator keeps the range relationship clear. Values can also be typed directly into the number.
Complete Frontmatter — All Controls Together editing state
---
title: Senior Software Engineer
company: Voi Technology ▼
website: voi.com
accent: acquired by Volvo Cars
location: Stockholm, Sweden
start: 2021-01 end: 2023-12
current: false
bullets: − 2 + to − 5 +
---
The principle: Every control lives inside the frontmatter block as a key: value pair. In view mode, all values render as plain monospace text — the entire block reads like YAML. In edit mode, each value type reveals its appropriate affordance (text input, dropdown, checkbox, stepper, date picker) while staying on the same line, in the same position, preserving the document flow. No field labels, no form layout — the YAML key IS the label. The frontmatter block IS the form.
