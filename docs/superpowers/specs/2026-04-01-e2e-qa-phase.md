# Spec: E2E QA Phase — Tiered Test Coverage

**Date:** 2026-04-01
**Status:** Draft

## Context

Multiple parallel sessions shipped changes across the entire stack: pipeline correctness (P1), archetype templates (P2), visual polish (P3), experience page redesign, and archetype content configuration. Before moving to the "Next" horizon (LLM enrichment, job-specific tailoring), we need confidence that the integrated system works end-to-end.

The project has Playwright + Testcontainers infrastructure (PR #33) with 10 existing tests across 3 suites (jobs, headlines, education). This QA phase fills the gaps.

## Existing Coverage

| Suite | File | Tests |
|---|---|---|
| Job list | `e2e/tests/jobs/job-list.spec.ts` | 3 tests — list, navigation, view switching |
| Headlines | `e2e/tests/resume/headlines.spec.ts` | 4 tests — CRUD + persistence |
| Education | `e2e/tests/resume/education.spec.ts` | 3 tests — CRUD + persistence |

## Test Plan — Two Tiers

### Tier 1: Smoke Tests (every page loads + basic interaction)

Quick tests that verify each page loads with seeded data and basic interactions work. Catches regressions from cross-cutting changes.

| Page | Route | Smoke Test |
|---|---|---|
| Profile | `/resume/profile` | Form loads with seeded data, edit a field, save, reload persists |
| Skills | `/resume/skills` | Categories load, create a category, add an item, delete |
| Archetypes list | `/archetypes/` | List loads with 5 archetypes, create one, delete it |
| Archetype detail | `/archetypes/$id` | Detail loads, headline picker works, tag weight slider adjusts |
| Job detail | `/jobs/$id` | Detail page loads with seeded job data |

### Tier 2: Deep Tests (critical paths)

Thorough coverage of the flows that directly serve the primary goal (resume generation per archetype).

#### Experience Page (redesigned)

| Test | What it verifies |
|---|---|
| Resume layout renders | Experiences display with title, company, dates, bullets — no card wrappers |
| Create experience | Dialog opens, fill form with MonthYearPicker, save, appears in list sorted by date |
| Edit experience | Gutter edit button opens dialog, change title, save, updated on page |
| Delete experience | Gutter delete button, confirmation dialog, experience removed |
| Inline bullet edit | Click `edit` pill on bullet, input appears, change text, save, text updated |
| Add bullet | Gutter "+ Add" button, inline input, enter text, bullet appears in list |
| Toggle variants | Click `⟳ N` pill, variant list expands below bullet with left border |
| Approve variant | PENDING variant shows ✓ pill, click it, status changes to APPROVED |
| Reject variant | PENDING variant shows ✗ pill, click it, status changes to REJECTED |
| Add variant | In expanded variant list, click "+ Add variant", fill form, variant appears |
| Delete variant | Click `del` pill on variant, variant removed |

#### Archetype Content Selection

| Test | What it verifies |
|---|---|
| Content selection loads | Archetype detail shows experience checkboxes with bullet variant pickers |
| Select experience | Check an experience, save, reload, selection persists |
| Select bullet variants | Expand experience, check specific variants, save, reload, selection persists |
| Select education | Check education entries, save, persist |
| Select skills | Check skill categories and items, save, persist |

#### Resume Generation

| Test | What it verifies |
|---|---|
| Generate resume for archetype | Select an archetype, hit generate on a job, PDF downloads (verify non-empty blob) |
| Different archetypes produce different PDFs | Generate for 2 different archetypes on same job, verify file sizes differ (content differs) |
| Content selection drives output | Generate PDF, verify it contains text from selected variants (PDF text extraction or filename check) |

## Parallel Session Split

Three sessions, each writing to separate test files under `e2e/tests/`:

### Session A: Smoke Tests

**Files:** `e2e/tests/resume/profile.spec.ts`, `e2e/tests/resume/skills.spec.ts`, `e2e/tests/archetypes/archetype-list.spec.ts`, `e2e/tests/archetypes/archetype-detail.spec.ts`, `e2e/tests/jobs/job-detail.spec.ts`

```
Read docs/superpowers/specs/2026-04-01-e2e-qa-phase.md (Tier 1 section). Read existing tests at e2e/tests/resume/headlines.spec.ts and e2e/tests/resume/education.spec.ts for patterns.

Write smoke tests for: profile, skills, archetypes list, archetype detail, and job detail pages. Each test should verify the page loads with seeded data and basic CRUD works.

Create a worktree:
git worktree add .claude/worktrees/e2e-smoke -b feat/e2e-smoke

Work inside that worktree. Run tests with: bun run --cwd e2e test
When done, run /land.
```
